# Guia do usuário

## O que o usuário faz

- acessar a VM disponibilizada;
- usar console Web ou SSH;
- instalar software autorizado de laboratório;
- executar experimentos;
- registrar resultados.

## O que o usuário não deve fazer

- alterar rede do cluster;
- apagar VMs, discos ou snapshots de terceiros;
- mexer em templates sem autorização;
- usar recursos do ambiente sem organização.

## Boas práticas

- documentar o que foi instalado;
- versionar scripts em Git, quando possível;
- encerrar a VM corretamente;
- avisar o administrador em caso de erro persistente.
